The Philosophy

Inspired by the analogy that "coding is like cooking," COOK lets you write programs that resemble recipes. No more worrying if your peeler supports carrots in version 4.3.

Selectively Interpreted

Built on Java, COOK tokenizes only the "Keys" (process and argument keys). Filler text, punctuation, and capitalization are often ignored, making it incredibly forgiving.

Architecture Detail

The Lexer selectively tokenizes lines based on defined keys. Then, the Classifier categorizes tokens, and the Interpreter executes the logic. Any text not matching a key is treated as a comment, allowing for "recipes" that read like plain English.

Key Ingredients

Forgiving Syntax

Add filler text, punctuation, or capitalization freely. Only the specific "Keys" matter to the compiler.

Statically-Typed

Despite the English-like appearance, the language maintains strict type safety under the hood for reliable execution.

Reference Keys

Variables and values are defined via special reference keys that point tokens, keeping the syntax clean.

Plain English

Designed for intuition. A COOK program reads like a list of instructions or a recipe card.
